# 附录:最小 Kafka + Spark Structured Streaming 实现思路

> 用于落地 2.7 节,使论文中"流处理框架"一章有对应代码与实验支撑。
> 目标:把"实时贷款申请"作为流式事件接入 Kafka,Spark Structured Streaming 消费后调用既有模型完成实时打分,结果回写 Kafka / 数据库,供前端看板订阅。

## 1. 总体拓扑

```
申请端(模拟器 / 前端表单)
        │  JSON 申请事件
        ▼
   Kafka Topic: loan-applications
        │
        ▼
Spark Structured Streaming(微批 5s)
   ├── 反序列化 + 特征工程(复用 features_v3.add_features)
   ├── 加载 default_model.joblib / fraud_model.joblib(广播变量)
   ├── 推理:P(违约) / P(欺诈) / 业务规则额度
   └── 输出:决策结果
        ▼
   Kafka Topic: loan-decisions       (前端 / 监控订阅)
   PostgreSQL: realtime_decisions    (历史记录,看板用)
```

## 2. 最小代码骨架(只列关键文件,不在此 commit 实现)

**`streaming/producer_sim.py`** — 模拟申请事件
```python
from kafka import KafkaProducer
import json, time, random, pandas as pd

producer = KafkaProducer(
    bootstrap_servers="localhost:9092",
    value_serializer=lambda v: json.dumps(v).encode("utf-8"),
)
sample = pd.read_csv("data/featured/train_repaired.csv").sample(1000)
for _, row in sample.iterrows():
    producer.send("loan-applications", row.to_dict())
    time.sleep(0.2)
```

**`streaming/spark_scorer.py`** — Structured Streaming 消费 + 打分
```python
from pyspark.sql import SparkSession
from pyspark.sql.functions import from_json, col, udf
from pyspark.sql.types import StructType, DoubleType
import joblib, pandas as pd
from features_v3 import add_features

spark = (SparkSession.builder
         .appName("loan-realtime-scorer")
         .config("spark.jars.packages",
                 "org.apache.spark:spark-sql-kafka-0-10_2.12:3.5.0")
         .getOrCreate())

# 广播模型(driver 加载一次,executor 复用)
default_bundle = joblib.load("artifacts/default_model.joblib")
fraud_bundle   = joblib.load("artifacts/fraud_model.joblib")
b_default = spark.sparkContext.broadcast(default_bundle)
b_fraud   = spark.sparkContext.broadcast(fraud_bundle)

raw = (spark.readStream
       .format("kafka")
       .option("kafka.bootstrap.servers", "localhost:9092")
       .option("subscribe", "loan-applications")
       .option("startingOffsets", "latest")
       .load())

schema = StructType()  # 与训练 schema 对齐,字段省略
events = raw.select(from_json(col("value").cast("string"), schema).alias("e")).select("e.*")

def score_partition(rows):
    """每个 partition 加载模型 → 批量打分,避免逐行反序列化。"""
    pdf = pd.DataFrame(list(rows))
    if pdf.empty:
        return iter([])
    feat = add_features(pdf)
    default_p = b_default.value["model"].predict(feat[b_default.value["feature_cols"]])
    fraud_p   = b_fraud.value["model"].predict_proba(feat[b_fraud.value["feature_cols"]])[:, 1]
    pdf["credit_score"] = default_p
    pdf["fraud_prob"]   = fraud_p
    return pdf.to_dict("records").__iter__()

scored = events.rdd.mapPartitions(score_partition).toDF()

(scored.writeStream
       .format("kafka")
       .option("kafka.bootstrap.servers", "localhost:9092")
       .option("topic", "loan-decisions")
       .option("checkpointLocation", "/tmp/ckpt/scorer")
       .outputMode("append")
       .start()
       .awaitTermination())
```

**`docker-compose.streaming.yml`** — 本地一键起 Kafka(单机演示足够)
```yaml
services:
  zookeeper:
    image: bitnami/zookeeper:3.9
    environment:
      ALLOW_ANONYMOUS_LOGIN: "yes"
  kafka:
    image: bitnami/kafka:3.6
    ports: ["9092:9092"]
    environment:
      KAFKA_CFG_ZOOKEEPER_CONNECT: zookeeper:2181
      KAFKA_CFG_LISTENERS: PLAINTEXT://:9092
      ALLOW_PLAINTEXT_LISTENER: "yes"
    depends_on: [zookeeper]
```

## 3. 论文里可以写的实验

| 实验 | 指标 | 用途 |
|---|---|---|
| 端到端延迟 | producer 发送 → Kafka 出口 → Spark 输出的 P50/P95(ms) | 证明实时性 |
| 吞吐量 | 单机 / 双 executor 下 events/sec | 证明可扩展性 |
| 批大小消融 | 微批触发间隔 1s / 5s / 10s 下的延迟–吞吐折中 | Structured Streaming 调优 |
| 在线打分一致性 | 同样输入下,流式打分结果 vs Flask 同步打分结果 的差异(MAE / AUC 差值) | 证明流式不损失精度 |

## 4. 与现有项目的衔接点

- **特征工程复用**:`features_v3.add_features()` 已是纯函数,可直接在 Spark UDF / mapPartitions 中调用。
- **模型制品复用**:`default_model.joblib` / `fraud_model.joblib` 是 sklearn 风格 bundle,Spark driver 加载后广播即可。
- **业务规则复用**:`service/flask/model_loader.py:_calculate_credit_limit` 可抽成纯函数,放到 streaming 包里同样调用。
- **看板对接**:`realtime_decisions` 表已存在(`stats.py` 中的 `fetch_recent_decisions`),Spark sink 直接 `foreachBatch` 写表即可,前端零改动。
